# 场景十：更新文件高亮状态

**前置条件**：分享盘下已存在文件，当前 `highlight` 为 `0`。

**步骤（高亮）**：
1. **[管理端]** 向 `PUT /api/admin/pans/:pan_id/files/:file_hash` 发送请求，请求体：
   ```json
   { "highlight": 1 }
   ```
2. **[管理端]** 断言响应状态码为 `200`。
3. **[管理端]** 调用 `GET /api/admin/pans/:pan_id` 确认该文件 `highlight` 已变为 `1`。
4. **[访客端]** 向 `GET /api/list` 获取文件列表，断言高亮文件排列在列表首位。

**步骤（取消高亮）**：
1. **[管理端]** 发送 `{ "highlight": 0 }`，断言 `200` 及状态恢复。
